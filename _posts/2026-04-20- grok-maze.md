# Creating a Procedural Maze Generator in UE5 Using Blueprints

Welcome to this beginner-friendly tutorial on building a fully procedural maze generator in Unreal Engine 5 using **only Blueprints**! 

By the end of this guide, you will have a reusable `BP_MazeGenerator` Actor that:
- Creates a perfect maze (no loops, every cell reachable) using the **Recursive Backtracker** algorithm.
- Uses a **MazeSeed** so you can recreate the exact same maze every time you play or reload the level.
- Generates **one maze per level**.
- Places **one random entrance** and **one random exit** on the outer perimeter (different cells, always on the border).
- Visualizes the maze efficiently with Hierarchical Instanced Static Mesh Components (HISM) so it performs well even at 20×20 grids.
- Runs in the editor (via Construction Script) **and** at runtime.

This is perfect for roguelikes, escape rooms, or any game that needs fresh mazes on demand.

**Prerequisites**
- Unreal Engine 5.3 or newer (tested on 5.4).
- Basic familiarity with Blueprints (variables, functions, Construction Script, arrays).
- A new **Blank** or **Third Person** project (we’ll keep it simple).
- Two simple Static Meshes (we’ll create placeholders if you don’t have any):
  - `SM_Floor` – a flat square plane (200×200 cm recommended).
  - `SM_Wall` – a thin vertical wall (200×20×200 cm, scaled later).

Let’s get started!

## Step 1: Project Setup and Asset Preparation

1. Create a new UE5 project (Blank template, no starter content needed).
2. In the **Content Browser**, create a folder called `MazeGenerator`.
3. Inside it, create two folders: `Blueprints` and `Meshes`.
4. (Optional but recommended) Create a simple floor and wall mesh:
   - Right-click → **Static Mesh** → create `SM_Floor` (use a scaled Cube or Plane).
   - Right-click → **Static Mesh** → create `SM_Wall` (thin vertical cube).

We will reference these later.

## Step 2: Creating the Data Structures

We need two custom structs to keep our maze data clean.

### 2.1 Create `S_NeighbourInfo`
1. Right-click in the `Blueprints` folder → **Blueprint Structure** → name it `S_NeighbourInfo`.
2. Add the following variables:
   - `CellIndex` (Integer)
   - `DeltaX` (Integer) – column offset
   - `DeltaY` (Integer) – row offset

<img src="{{ '/assets/images/blog/Part1-Step-4.png' | relative_url }}" style="width:100%;" alt="The completed S_NeighbourInfo struct with CellIndex, DeltaX, and DeltaY variables." class="post-image">

### 2.2 Create `S_MazeCell`
1. Right-click → **Blueprint Structure** → name it `S_MazeCell`.
2. Add the following variables:
   - `Row` (Integer)
   - `Col` (Integer)
   - `bVisited` (Boolean, default False)
   - `bWallNorth` (Boolean, default True)
   - `bWallEast` (Boolean, default True)
   - `bWallSouth` (Boolean, default True)
   - `bWallWest` (Boolean, default True)

This struct represents one cell in the grid and tracks its four walls.

## Step 3: Create the Maze Generator Blueprint

1. Right-click in `Blueprints` → **Blueprint Class** → **Actor** → name it `BP_MazeGenerator`.
2. Open `BP_MazeGenerator`.

## Step 4: Add Variables and Components

In the **My Blueprint** panel (left side):

### Variables (all Instance Editable unless noted):
- `MazeWidth` (Integer, Default 12, Min 5, Max 30)
- `MazeHeight` (Integer, Default 12, Min 5, Max 30)
- `CellSize` (Float, Default 200.0)
- `MazeSeed` (Integer, Default 42) – change this to get a new maze!
- `MazeGrid` (Array of `S_MazeCell`) – our 1D grid storage
- `RandomStream` (Random Stream) – seeded random source
- `FloorHISM` (Hierarchical Instanced Static Mesh Component) – will hold all floors
- `WallHISM` (Hierarchical Instanced Static Mesh Component) – will hold all walls

### Components:
1. In the **Components** panel, click **Add** → **Hierarchical Instanced Static Mesh** → name it `FloorHISM`.
2. Add another `Hierarchical Instanced Static Mesh` → name it `WallHISM`.
3. Select `FloorHISM` → Details → **Static Mesh** → assign your `SM_Floor`.
4. Select `WallHISM` → Details → **Static Mesh** → assign your `SM_Wall`.

<img src="{{ '/assets/images/blog/Part1-Step-8.png' | relative_url }}" style="width:100%;" alt="BP_MazeGenerator variables and HISM components set up with default values." class="post-image">

## Step 5: Implement Maze Generation (Construction Script)

We will generate the maze in the **Construction Script** so you can see it instantly in the editor.

### 5.1 Clear Previous Data
Drag a **Construction Script** node.

- From it, call **Clear Instances** on both `FloorHISM` and `WallHISM`.
- Set `MazeGrid` to an empty array (use **Clear** node).

### 5.2 Initialize the Random Stream
- Drag your `MazeSeed` variable → **Make RandomStream** node (Initial Seed pin).
- Set the result into the `RandomStream` variable.

<img src="{{ '/assets/images/blog/Part1-Step-10.png' | relative_url }}" style="width:100%;" alt="Construction Script initializing the seeded RandomStream." class="post-image">

### 5.3 Create the Empty Grid
Create a custom **Function** called `InitializeGrid` (Pure = false).

Inside it:
1. Clear `MazeGrid`.
2. For Loop (First Index 0, Last Index = MazeWidth × MazeHeight – 1):
   - Create a new `S_MazeCell`.
   - Set its `Row` = Index ÷ MazeWidth (integer division).
   - Set its `Col` = Index % MazeWidth.
   - Set all four `bWall*` to **True**.
   - Add the cell to `MazeGrid`.

Call this function from Construction Script.

### 5.4 The Recursive Backtracker Algorithm

Create another custom **Function** called `GenerateMaze` (call it after `InitializeGrid`).

We use the **iterative** version with a stack (avoids Blueprint recursion depth issues).

**High-level logic (implemented in Blueprints):**
1. Pick a random starting cell index using `Random Integer in Range from Stream` (0 to MazeWidth×MazeHeight-1).
2. Mark it visited.
3. Push it onto a local **TArray<int32>** called `Stack`.
4. While the stack is **not empty**:
   - Get current cell index from top of stack.
   - Get list of unvisited neighbors (we’ll create a helper for this).
   - If neighbors exist:
     - Pick one at random (using our seeded stream).
     - Remove the wall between current and neighbor (update both cells).
     - Mark neighbor visited.
     - Push neighbor onto stack.
   - Else:
     - Pop the current cell from stack.

#### Helper Function: `GetUnvisitedNeighbours`
Create a pure **Function** `GetUnvisitedNeighbours` that takes `CurrentIndex` (int) and returns `Array of S_NeighbourInfo`.

Inside:
- Calculate Row/Col from index.
- Create four `S_NeighbourInfo` structs for N/E/S/W:
  - North: CellIndex = CurrentIndex – MazeWidth, DeltaX=0, DeltaY=-1
  - East:  CellIndex = CurrentIndex + 1, DeltaX=1, DeltaY=0
  - South: CellIndex = CurrentIndex + MazeWidth, DeltaX=0, DeltaY=1
  - West:  CellIndex = CurrentIndex – 1, DeltaX=-1, DeltaY=0
- For each, check bounds and `!MazeGrid[CellIndex].bVisited`.
- Add valid ones to the return array.

<img src="{{ '/assets/images/blog/Part1-Step-15.png' | relative_url }}" style="width:100%;" alt="GetUnvisitedNeighbours function returning valid neighbour structs." class="post-image">

#### Helper Function: `RemoveWallBetween`
Create **Function** `RemoveWallBetween` (CurrentIndex, NeighbourIndex, DeltaX, DeltaY).

Based on the Delta values, set the appropriate `bWall*` to **False** on **both** cells.

Example (North):
- If DeltaY == -1 → Current cell: bWallNorth = false; Neighbour cell: bWallSouth = false.

Do the same for East/South/West.

<img src="{{ '/assets/images/blog/Part1-Step-17.png' | relative_url }}" style="width:100%;" alt="RemoveWallBetween function updating both cells' walls." class="post-image">

## Step 6: Rendering the Maze

Create a custom **Function** `BuildMazeVisuals` (call at the very end of Construction Script).

Inside:
1. **Floors**:
   - For each cell in `MazeGrid`:
     - Calculate world location: `FVector(Col * CellSize, Row * CellSize, 0)`
     - Add Instance to `FloorHISM` with that transform (scale 1, rotation 0).

2. **Walls**:
   - For each cell:
     - Calculate base location.
     - If `bWallNorth` = true → add wall instance at offset (0, -CellSize/2, CellSize/2), rotation Yaw=0.
     - If `bWallEast` = true → add wall instance at offset (CellSize/2, 0, CellSize/2), rotation Yaw=90.
     - If `bWallSouth` = true → add wall instance at offset (0, CellSize/2, CellSize/2), rotation Yaw=180.
     - If `bWallWest` = true → add wall instance at offset (-CellSize/2, 0, CellSize/2), rotation Yaw=-90.

Use **Add Instance** node on the appropriate HISM with a **Make Transform** for each wall.

<img src="{{ '/assets/images/blog/Part1-Step-20.png' | relative_url }}" style="width:100%;" alt="BuildMazeVisuals function adding floor and wall instances with correct transforms." class="post-image">

## Step 7: Adding Random Entrance and Exit

Create **Function** `CreateEntranceAndExit`.

Call it **after** `GenerateMaze` but **before** `BuildMazeVisuals`.

Logic:
1. Build a local array of all border cell indices (Row==0 or Row==MazeHeight-1 or Col==0 or Col==MazeWidth-1).
2. Use the seeded `RandomStream` to shuffle or pick two **different** random indices from the border array → EntranceIndex and ExitIndex.
3. Open the outer wall for each:
   - If the cell is on the North border (Row==0) → set `bWallNorth = false`
   - South border → `bWallSouth = false`
   - West → `bWallWest = false`
   - East → `bWallEast = false`

Because we open the outer wall, the maze now has two clear entrances/exits to the outside world.

<img src="{{ '/assets/images/blog/Part1-Step-23.png' | relative_url }}" style="width:100%;" alt="CreateEntranceAndExit function selecting random border cells and removing outer walls." class="post-image">

(Optional) Spawn a `PlayerStart` actor at the entrance cell center if you want automatic player placement:
- Use `Spawn Actor from Class` with your PlayerStart class and the entrance world location.

## Step 8: Final Construction Script Flow & Testing

Your Construction Script should now call these functions **in order**:
1. Clear instances & grid
2. Initialize Random Stream
3. InitializeGrid
4. GenerateMaze
5. CreateEntranceAndExit
6. BuildMazeVisuals

Drag your `BP_MazeGenerator` into any level.  
Change the **MazeSeed** in the Details panel → the maze instantly updates in the editor!  
Play the level → you get the exact same maze every time.

## Bonus Tips for Beginners

- **Performance**: 20×20 grid = 400 cells → still runs instantly. For larger mazes, consider async generation.
- **Collision**: Make sure your `SM_Wall` has collision enabled.
- **Different Seeds**: Just change the `MazeSeed` value – every integer gives a unique reproducible maze.
- **Extending**: Add doors at entrance/exit, collectibles inside cells, or pathfinding (A* on the same grid data).
- **Debug**: Temporarily add `Print String` nodes showing current index and visited status.
